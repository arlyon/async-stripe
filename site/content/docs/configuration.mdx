**async-stripe** is runtime-agnostic but provides convenient defaults. You must select a runtime feature flag in the `async-stripe` crate.

## Runtime Selection

- **`runtime-tokio-hyper`**: *(Recommended)* Uses `tokio` and `hyper`. Best for general-purpose async applications.
- **`runtime-async-std-surf`**: Uses `async-std` and `surf`.
- **`blocking`**: A synchronous blocking client. Note that this wraps Tokio internally to provide a synchronous API.

## Request Timeouts

Use `.timeout(Duration)` on `ClientBuilder` to set a default timeout for each HTTP attempt. You can also call `.timeout(...)` on an individual customizable request to override that client default for a single API call. If you also configure retries, each retry gets a fresh timeout budget and backoff sleeps are not counted toward that limit.

For the Hyper async client, the default is no timeout unless you set one. For the blocking client built with `build_sync()`, async-stripe applies a default per-attempt timeout of 30 seconds so synchronous calls are never unbounded.
